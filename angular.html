What are the Angular component lifecycle hooks and in what order do they run?
Medium▼
HookWhenUse for
ngOnChangesBefore ngOnInit, then whenever @Input changesReact to parent input changes; SimpleChanges holds prev/current
ngOnInitOnce, after first ngOnChangesComponent initialization, HTTP calls, set up subscriptions
ngDoCheckEvery change detection runCustom change detection (avoid — expensive)
ngAfterContentInitOnce, after ng-content projectedAccess ContentChild/ContentChildren
ngAfterContentCheckedAfter each check of projected contentRarely needed
ngAfterViewInitOnce, after component view is initialisedAccess ViewChild/ViewChildren; interact with DOM
ngAfterViewCheckedAfter every view checkRarely needed; avoid modifying state here
ngOnDestroyJust before component is destroyedUnsubscribe, clear timers, cancel HTTP requests
TypeScript
@Component({ selector: 'app-user', template: '...' })
export class UserComponent implements OnInit, OnDestroy {
  @Input() userId!: number;
  private sub!: Subscription;

  ngOnInit() {
    // safe to use userId here — inputs are set by this point
    this.sub = this.userService.get(this.userId).subscribe(...);
  }

  ngOnDestroy() {
    this.sub.unsubscribe(); // prevent memory leak
  }
}

RxJS & Observables

Operators, subjects, error handling, patterns

5
Explain the difference between map, switchMap, mergeMap, concatMap, and exhaustMap.
Hard▼

All four flatten an Observable-of-Observables, but they differ in how they handle concurrent inner observables:

OperatorConcurrencyUse case
mapn/a (synchronous transform)Transform each value — no inner Observable
switchMapCancels previous inner obs on new emissionTypeahead search — only care about latest request
mergeMapAll inner obs run concurrentlyUpload multiple files in parallel
concatMapQueues — waits for previous to completeSequential actions where order matters (shopping cart steps)
exhaustMapIgnores new emissions while inner obs is activeLogin button — ignore re-clicks while request is in flight
TypeScript
// Typeahead — switchMap cancels stale requests
this.searchControl.valueChanges.pipe(
  debounceTime(300),
  distinctUntilChanged(),
  switchMap(term => this.api.search(term))  // cancels previous HTTP call
).subscribe(results => this.results = results);

// Login button — exhaustMap ignores double-click
this.loginClick$.pipe(
  exhaustMap(() => this.authService.login(this.form.value))
).subscribe(...);

What is the difference between Subject, BehaviorSubject, ReplaySubject, and AsyncSubject?
Hard▼
TypeInitial value?Late subscribers get?Use when
SubjectNoNothing (miss past values)Events (button clicks, notifications)
BehaviorSubjectYes (required)Current value immediatelyState (user session, theme, cart count)
ReplaySubject(n)NoLast n values replayedCaching N last emissions for late subscribers
AsyncSubjectNoOnly the last value, on completeAsync operations that emit one final result
TypeScript
// BehaviorSubject for state — common pattern in services
@Injectable({ providedIn: 'root' })
export class CartService {
  private items$ = new BehaviorSubject<CartItem[]>([]);
  readonly items = this.items$.asObservable(); // expose read-only

  addItem(item: CartItem) {
    this.items$.next([...this.items$.getValue(), item]);
  }
}

// Component subscribes and always gets current state immediately
cartService.items.subscribe(items => this.cartItems = items);
7
How do you handle errors in RxJS? What is the difference between catchError, retry, and retryWhen?
Medium▼
TypeScript
// catchError — recover and return fallback or rethrow
this.http.get<User[]>('/api/users').pipe(
  catchError(err => {
    if (err.status === 404) return of([]);          // return empty array
    return throwError(() => new Error(err.message)); // rethrow
  })
);

// retry(n) — resubscribes n times on error (no delay)
this.api.get().pipe(retry(3)).subscribe(...);

// retryWhen with exponential backoff (RxJS 6)
this.api.get().pipe(
  retryWhen(errors$ => errors$.pipe(
    scan((count, err) => {
      if (count >= 3) throw err; // give up after 3
      return count + 1;
    }, 0),
    delayWhen(count => timer(count * 1000)) // 1s, 2s, 3s delay
  ))
);

How do you prevent memory leaks from subscriptions? Compare takeUntil, async pipe, and takeUntilDestroyed.
Hard▼
TypeScript
// Option 1: async pipe — cleanest, auto-unsubscribes, triggers CD
// template: {{ users$ | async }}
users$ = this.userService.getAll(); // no subscribe needed

// Option 2: takeUntilDestroyed (Angular 16+) — inject in constructor
private destroyRef = inject(DestroyRef);

ngOnInit() {
  this.data$.pipe(
    takeUntilDestroyed(this.destroyRef)
  ).subscribe(data => this.data = data);
}

// Option 3: takeUntil pattern (pre Angular 16)
private destroy$ = new Subject<void>();

ngOnInit() {
  interval(1000).pipe(takeUntil(this.destroy$)).subscribe(...);
}
ngOnDestroy() { this.destroy$.next(); this.destroy$.complete(); }
💡
Modern best practice
In Angular 16+ with standalone components, prefer takeUntilDestroyed() — it's the cleanest pattern. async pipe is best for templates. Avoid manual Subscription arrays.

Change Detection

Zone.js, OnPush, Signals

9
How does Angular's change detection work? What is Zone.js's role?
Hard▼

Angular uses Zone.js which monkey-patches async APIs (setTimeout, Promise, XHR, DOM events). When any async task completes, Zone.js notifies Angular, which then runs change detection from the root component down the tree — comparing current and previous template expressions. Any change triggers DOM updates.

Default strategy — checks every component, every time. Simple but can be slow with large trees.

OnPush strategy — component is checked only when:

  • An @Input() reference changes (not mutation)
  • An event originates from the component or its children
  • An Observable used with async pipe emits
  • You manually call markForCheck() or detectChanges()
TypeScript
@Component({
  selector: 'app-list',
  changeDetection: ChangeDetectionStrategy.OnPush, // opt-in to OnPush
  template: `<li *ngFor="let item of items">{{ item.name }}</li>`
})
export class ListComponent {
  @Input() items: Item[] = [];
  // Must pass new array reference to trigger re-check:
  // ✅ this.items = [...this.items, newItem]
  // 🚫 this.items.push(newItem)  ← doesn't trigger OnPush
}
10
What are Angular Signals (Angular 16+) and how do they differ from traditional change detection?
Hard▼

Signals are reactive primitives that track their value and notify consumers when it changes — without Zone.js. Angular can update only the exact DOM nodes affected by a changed signal, enabling truly granular updates.

TypeScript
@Component({
  template: `
    <p>Count: {{ count() }}</p>          <!-- signal read = function call -->
    <p>Double: {{ doubled() }}</p>
    <button (click)="increment()">+</button>
  `
})
export class CounterComponent {
  count   = signal(0);                          // writable signal
  doubled = computed(() => this.count() * 2); // derived, auto-updates

  increment() { this.count.update(v => v + 1); }

  // effect() — side effects when signals change
  constructor() {
    effect(() => console.log('Count changed:', this.count()));
  }
}

// Convert Observable to Signal
users = toSignal(this.userService.getAll(), { initialValue: [] });
ℹ️
Direction of Angular
Angular is moving toward a Zoneless future — Signals + provideExperimentalZonelessChangeDetection() (Angular 17+). Knowing Signals is increasingly important in interviews for Angular 16+ shops.

Routing

Lazy loading, guards, resolvers

11
How does lazy loading work in Angular? What is a Route Guard?
Medium▼
TypeScript
// Lazy loading — loadComponent/loadChildren defer the bundle
const routes: Routes = [
  {
    path: 'admin',
    canActivate: [authGuard],     // guard runs before loading
    loadChildren: () => import('./admin/admin.routes')
                        .then(m => m.ADMIN_ROUTES)
  },
  {
    path: 'profile',
    loadComponent: () => import('./profile/profile.component')
                         .then(m => m.ProfileComponent),
    resolve: { user: userResolver }  // pre-fetch data before activation
  }
];

// Functional guard (Angular 15+ — preferred over class guards)
export const authGuard: CanActivateFn = (route, state) => {
  const auth = inject(AuthService);
  const router = inject(Router);
  return auth.isLoggedIn()
    ? true
    : router.createUrlTree(['/login'], { queryParams: { returnUrl: state.url } });
};

// Functional resolver — pre-fetches data before component activates
export const userResolver: ResolveFn<User> = (route) => {
  return inject(UserService).findById(+route.paramMap.get('id')!);
};

What is the difference between ActivatedRoute params, queryParams, and data? How do you read route parameters?
Easy▼
TypeScript
// Route: /products/:id?filter=active
@Component({...})
export class ProductComponent {
  private route = inject(ActivatedRoute);

  constructor() {
    // params — path variables (:id) — Observable for dynamic changes
    this.route.params.subscribe(p => this.productId = +p['id']);

    // queryParams — ?filter=active
    this.route.queryParamMap.subscribe(q => this.filter = q.get('filter'));

    // data — static data from route config or resolver result
    this.route.data.subscribe(d => this.user = d['user']); // from resolver

    // Snapshot — use only if you know param won't change during component lifetime
    const id = this.route.snapshot.paramMap.get('id');
  }
}

How do you implement a custom async validator (e.g. check if email is unique)?
Medium▼
TypeScript
// Functional async validator — preferred (Angular 14+)
export function uniqueEmailValidator(userService: UserService): AsyncValidatorFn {
  return (control: AbstractControl): Observable<ValidationErrors | null> => {
    return userService.checkEmail(control.value).pipe(
      debounceTime(300),         // don't fire on every keystroke
      switchMap(isTaken =>
        isTaken ? of({ emailTaken: true }) : of(null)
      )
    );
  };
}

// Usage in FormBuilder — 3rd param is async validators array
email: ['',
  [Validators.required, Validators.email],
  [uniqueEmailValidator(this.userService)]
]

<!-- Template: show error during/after async check -->
<mat-error *ngIf="form.get('email')?.errors?.['emailTaken']">
  Email is already registered
</mat-error>
<mat-spinner *ngIf="form.get('email')?.pending" diameter="16"></mat-spinner>

Performance & Best Practices

Bundle size, lazy loading, trackBy, SSR

15
How do you optimise Angular application performance? List at least 5 techniques.
Hard▼
  1. OnPush change detection — reduces the number of component checks; combine with immutable data
  2. Lazy load routes — split code by route so initial bundle is small
  3. trackBy in *ngFor — prevents Angular from destroying and re-creating DOM elements on list update
  4. Signals (Angular 16+) — fine-grained reactivity without Zone.js overhead
  5. Defer blocks (Angular 17) — @defer lazily loads component when it enters the viewport
  6. async pipe — auto-unsubscribes, triggers CD precisely when needed
  7. Pure pipes — cached; only re-runs when input reference changes
  8. Bundle optimisation — Angular's esbuild builder (default in v17), tree-shaking, removing unused imports
HTML
<!-- trackBy prevents full list re-render -->
<li *ngFor="let user of users; trackBy: trackByUserId">
  {{ user.name }}
</li>

<!-- @defer (Angular 17) — loads when in viewport -->
@defer (on viewport) {
  <app-heavy-chart [data]="chartData" />
} @placeholder {
  <div class="skeleton"></div>
}
TypeScript
trackByUserId(index: number, user: User) { return user.id; }
16
What is Angular Universal (SSR)? What problems does it solve?
Medium▼

Angular Universal renders the app on a Node.js server, sending fully-formed HTML to the client. Solves:

  • SEO — search crawlers see complete HTML instead of an empty shell
  • First Contentful Paint (FCP) — user sees content before JavaScript executes
  • Social sharing — Open Graph / Twitter Card meta tags are in the initial response

Hydration (Angular 17) — the server renders HTML; the client "hydrates" by attaching event listeners without re-rendering from scratch, eliminating the flash of blank content.

⚠️
SSR gotchas
No window, document, or localStorage on the server. Wrap browser-only code with isPlatformBrowser(). HTTP calls made during SSR are cached via withHttpTransferCache() so the client doesn't re-fetch on hydration.

19
What is the difference between a structural directive (*ngIf, *ngFor) and an attribute directive?
Easy▼

Structural directives — change the DOM structure by adding/removing elements. Prefixed with * (syntactic sugar for <ng-template>). Examples: *ngIf, *ngFor, *ngSwitch. Angular 17+ offers built-in control flow: @if, @for, @switch — faster and no import needed.

Attribute directives — modify the appearance or behaviour of an existing element without changing DOM structure. Examples: NgClass, NgStyle, custom tooltip directive.

TypeScript
// Custom attribute directive — highlight on hover
@Directive({ selector: '[appHighlight]', standalone: true })
export class HighlightDirective {
  @Input('appHighlight') color = 'yellow';

  constructor(private el: ElementRef) {}

  @HostListener('mouseenter') onEnter() {
    this.el.nativeElement.style.backgroundColor = this.color;
  }
  @HostListener('mouseleave') onLeave() {
    this.el.nativeElement.style.backgroundColor = '';
  }
}
<!-- Usage: <p appHighlight="lightblue">Hover me</p> -->

How do you implement HTTP interceptors in Angular? What common use cases do they solve?
Medium▼
TypeScript
// Functional interceptor (Angular 15+ — preferred)
export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const token = inject(AuthService).getToken();
  if (!token) return next(req);

  const authReq = req.clone({
    headers: req.headers.set('Authorization', `Bearer ${token}`)
  });
  return next(authReq).pipe(
    catchError(err => {
      if (err.status === 401) inject(AuthService).logout();
      return throwError(() => err);
    })
  );
};

// Register in bootstrapApplication
provideHttpClient(withInterceptors([authInterceptor, loggingInterceptor]))

Common interceptor use cases: attach auth token, retry on 503, show/hide global loading spinner, log all requests, add correlation ID header, handle 401 token refresh.
